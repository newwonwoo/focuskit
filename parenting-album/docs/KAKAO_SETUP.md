# 카카오 OpenBuilder 설정 가이드

아빠·엄마·조부모 등 가족이 **카카오톡 채널**을 통해 사진/영상/메모를 봇에게 전송할 수 있도록 설정합니다.

> 예상 소요 시간: **30~45분** (카카오 가입·채널 개설·스킬 설정 포함)
> 전제 조건: Vercel 배포가 완료되어 Webhook URL이 준비되어 있어야 합니다. ([VERCEL_SETUP.md](./VERCEL_SETUP.md) 참고)

---

## 0. 개념 이해 (1분)

```
 카카오톡 앱
   │
   ▼  (사용자가 "원우 앨범봇" 채널 친구 추가 → 1:1 대화)
 카카오톡 채널
   │
   ▼  (카카오 Business / 채널 관리자 센터)
 Kakao i OpenBuilder (챗봇 빌더)
   │
   ▼  스킬(Skill) = 외부 서버 호출
 Vercel Webhook (우리가 만든 api/kakao/webhook.ts)
```

**무료 범위**: 챗봇 응답, 스킬 Webhook 호출 모두 무료. 메시지 수 제한 없음.
(알림톡/친구톡 같은 마케팅 발송은 유료지만, 봇 응답은 해당 없음.)

## 1. 카카오 비즈니스 계정 개설

1. https://business.kakao.com 접속
2. 카카오 계정으로 로그인
3. 본인 인증 (휴대폰 인증 1회)
4. **개인 자격**으로도 채널 개설 가능 — 사업자 등록 불필요

---

## 2. 카카오톡 채널 만들기

1. https://center-pf.kakao.com 접속 → [채널 만들기] 클릭
2. 폼 작성:
   - **채널 이름**: `원우 앨범봇`
   - **검색용 아이디**: `wonwoo-album` (유니크 ID, 중복 안 되는 값)
   - **카테고리**: 기타 > 개인 / 커뮤니티
   - **프로필 사진**: 원우 사진이나 귀여운 일러스트 (선택)
3. [완료] → 채널 개설
4. 상단의 채널 URL (예: `https://pf.kakao.com/_ABCDEF`) 를 저장 — 가족에게 공유할 초대장입니다.

### 노출 설정 (중요)
- 좌측 메뉴 → **설정** → **채널 노출** → `검색 허용: OFF` 로 설정
- 이유: 아무나 검색해서 찾지 못하게 (가족만 링크로 접근)

---

## 3. OpenBuilder 봇 생성

1. https://i.kakao.com 접속
2. 로그인 후 [내 챗봇] → [+ 챗봇 만들기] 클릭
3. 챗봇 유형: **카카오톡 채널**
4. 연결할 채널: **원우 앨범봇** 선택
5. 봇 이름: `wonwoo-album-bot`
6. [생성] 클릭

---

## 4. 스킬 등록 (Webhook 연결)

### 4-1. 스킬 만들기

1. 좌측 메뉴 → **스킬** → [+ 생성]
2. 스킬 이름: `save-moment`
3. **URL**: Vercel에 배포된 Webhook 주소 + secret 쿼리파라미터
   ```
   https://your-project.vercel.app/api/kakao/webhook?secret=실제시크릿값
   ```
   - `실제시크릿값`은 Vercel 환경변수 `KAKAO_WEBHOOK_SECRET`에 설정한 값과 동일해야 함
4. **HTTP Method**: POST
5. **응답 타입**: Skill Response
6. [저장]

### 4-2. 파라미터 설정

기본적으로 Kakao는 사진을 보내지 않으면 스킬에 이미지 URL을 실어주지 않습니다. 사진 수신을 활성화하려면:

1. 좌측 메뉴 → **엔티티** → [+ 엔티티 추가]
2. 엔티티 이름: `media_url`
3. 타입: `sys.any` (또는 URL 타입)
4. 저장

스킬 편집 화면으로 돌아가서:
1. 파라미터 섹션에서 `media_url` 매핑
2. Kakao가 자동으로 `action.params.media_url` 또는 `action.detailParams.media_url.value`에 URL을 담아줌

> 💡 **실전 팁**: Kakao OpenBuilder의 이미지 수신은 설정이 미묘합니다. 처음엔 텍스트만 먼저 돌아가는 것을 확인한 뒤, 이미지 수신을 단계적으로 추가하는 게 편합니다.

## 5. 블록 등록 (언제 스킬이 실행되는지)

### 5-1. 기본 폴백 블록

1. 좌측 **시나리오** → 기본 시나리오 선택
2. **폴백 블록**(Fallback Block) 을 수정:
   - 사용자가 어떤 메시지를 보내든 이 블록으로 라우팅되게 만듭니다
3. 블록 안에 **스킬 실행** 노드 추가 → `save-moment` 스킬 선택
4. 응답 타입: **스킬데이터 사용** (스킬이 반환한 simpleText를 그대로 사용자에게 보여줌)

### 5-2. 웰컴 블록 비활성화 (선택)

- 기본 웰컴 블록("안녕하세요! 저는...")은 우리 봇의 첫 인사와 충돌할 수 있으니 비활성화 또는 심플하게 수정

---

## 6. 배포 테스트

1. OpenBuilder 우측 상단 [배포] 클릭
2. **테스트 봇 실행** → 오픈빌더 내장 채팅 UI에서 "안녕" 입력
3. 기대 결과: `안녕하세요! 원우 앨범봇이에요 🌷 / 원우와 어떤 관계이신가요?`
4. "아빠" 입력 → `아빠로 등록 완료! 🎉`
5. 사진 전송 → `아빠의 기록 저장 중이야 📸`

### Vercel 로그 확인
- Vercel 대시보드 → 프로젝트 → Functions → Logs
- `[webhook] new user created`, `[webhook bg] media entry saved` 같은 로그가 찍히면 성공
- Notion DB에도 실제로 row가 추가되는지 확인

## 8. 트러블슈팅

### "챗봇이 응답하지 않습니다" 표시
- Vercel Webhook URL이 잘못됐거나 5초 내 응답 못 함
- Vercel 로그 확인 → 어디서 막혔는지 파악
- 본 프로젝트는 즉시 응답 + waitUntil 백그라운드 패턴이라 5초 내 반드시 응답함

### "권한 오류 (401)"
- URL 쿼리의 `secret` 값이 Vercel 환경변수와 다름
- OpenBuilder의 스킬 URL에서 `secret=` 뒤의 값 재확인

### 사진이 Notion에 안 쌓임
- Kakao가 실제로 이미지 URL을 넘겨주는지 Vercel 로그에서 `payload.mediaUrls` 확인
- 로그에 빈 배열이면 OpenBuilder의 엔티티/파라미터 설정 문제
- `lib/kakao.ts`의 `collectUrlStrings`가 모든 경로를 탐색하므로 URL이 payload 어디엔가 있으면 잡힙니다

### 같은 메시지가 중복 저장됨
- idempotency key는 `userId + timestamp + utterance + mediaSignature` 조합
- 동일 순간에 2번 보내면 key가 같아 중복 방지됨
- 서로 다른 타임스탬프면 다른 key로 저장되는 게 정상

### 이미 등록된 가족인데 다시 등록 질문 나옴
- Users DB의 `state` 컬럼이 `awaiting_name`인지 확인 → `active`로 수정
- Notion 캐시는 60초 TTL이므로 잠시 후 반영됨
